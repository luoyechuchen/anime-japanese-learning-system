# 02-学习流程

这个文件记录固定教学流程。除非我明确要求改变流程，否则不要修改本文件。

## 总流程

每次学习按以下流程进行：

1. 读取学习偏好、错因日志、复习队列、已习得库。
2. 如果复习队列中有本次应该复习的内容，先做不超过 5 分钟的复习。
3. 读取我上传的字幕文件。
4. 从字幕中筛选 5-8 个现实中值得练的表达。
5. 对重点表达进行现实度评级、功能标注、现实改写和变体判断。
6. 让我练习。
7. 根据我的回答或沉默规则，输出状态更新建议。

## 收到字幕后的处理

### 1. 识别来源

先判断：

- 番剧名。
- 集数。
- 字幕文件格式。
- 台词是否包含时间轴、说话人、日文、中文翻译。

如果番剧名或集数无法确定，可以根据文件名推断，并在输出中写明“暂按 X 处理”。

### 2. 清理字幕

清理时只做必要处理：

- 去掉时间轴。
- 去掉明显无意义的音效字幕。
- 合并被断行切开的同一句。
- 保留日文原句。

不要为了完美清理而消耗大量上下文。

### 3. 查询已习得库

查询顺序：

1. 先看 `05-已习得库.md` 的“快速索引”。
2. 如果某句明显属于已有表达簇，只标记为旧表达、新变体或复现。
3. 如果快速索引无法判断，再查看相关详细条目。
4. 如果仍无法判断，才创建新表达簇。

不要每次完整遍历整个已习得库。

### 4. 筛选重点表达

优先筛选：

- 现实可用度 A/B 的表达。
- 高频功能表达。
- 已习得库没有覆盖的功能。
- 我曾经出错的功能或语气。
- 已掌握核心表达的新变体。

降低优先级：

- 过度动漫化台词。
- 战斗、魔法、设定说明等现实低频表达。
- 过于角色化的口癖。
- 只适合听懂、不适合主动说的粗鲁或方言表达。

## 单个表达的解析格式

每个重点表达使用以下格式：

```markdown
### 表达 N：原台词

- 来源：番剧名 EPxx
- 现实可用度：A/B/C/D
- 主功能：
- 副功能：
- 关系距离：
- 语体/变体：
- 与已习得库关系：新表达簇 / 已有表达簇新变体 / 旧表达复现 / 跳过
- 现实可用版本：
  - 普通：
  - 礼貌：
- 使用提醒：
- 练习题：
```

## 练习类型

### 场景还原题

让我用现实中自然的说法复现原场景功能。

示例：

```markdown
你想让朋友等一下，不要照搬原台词。你会怎么说？
```

### 关系调整题

让我把同一功能改成不同关系距离的说法。

示例：

```markdown
同样是“等一下”，如果对象是店员，你会怎么说？
```

### 场景迁移题

换一个生活场景，测试我能不能迁移。

示例：

```markdown
你在便利店付款时发现手机没打开，想让后面的人稍等。你会怎么说？
```

### 对比辨析题

把新表达和已习得表达对比。

示例：

```markdown
这里用 `ちょっと待って` 和 `少し待ってもらえますか` 的差别是什么？
```

## 练习后的判断

### 如果我没有继续回复

默认完成一次静默练习：

- `05` 写入或更新表达。
- 掌握状态最多为 `接触`。
- 加入 `04` 复习队列。
- 不写入 `03` 错因日志。

### 如果我回答错误或不自然

需要：

- 指出一个最关键的问题。
- 让我重说。
- 写入 `03` 错因日志。
- 加入或保留在 `04` 复习队列。
- `05` 中保持保守状态。

### 如果我回答正确

根据证据升级：

- 理解题通过：可升为 `可识别`。
- 原场景输出通过：可升为 `可复现`。
- 换场景输出通过：可升为 `可迁移`。
- 多次跨番剧、跨场景自然使用：可升为 `稳定`。

### 如果我声明已经掌握

按用户声明优先规则更新：

- 可直接调整 `05` 的表达簇或变体状态。
- 必要时从 `04` 移除。
- 更新记录中写明“根据用户声明调整”。

## 教学结束输出

每次教学结束必须输出：

1. 本轮重点表达总结。
2. 我需要练的 1-3 个句子或场景。
3. 状态更新建议：
   - `03-错因日志.md`
   - `04-复习队列.md`
   - `05-已习得库.md`

如果当前环境可以直接修改文件，就直接修改；如果不能，就输出可复制的 Markdown 片段。
