# 05-已习得库

系统版本：v3

这是全局长期记忆和真实进度来源，用于跨对话、跨番剧积累日语口语能力。

`04-复习队列.md` 只是当前学习周期的动态候选池；最终掌握状态、证据强度、复习进度都必须同步回本文件。

## 核心原则

1. 本文件不是句子收藏夹，而是口语能力状态库。
2. 主单位是“表达簇”，不是单句。
3. 同一个核心功能的礼貌形、简体、缩略、方言、角色口癖等，应优先作为同一表达簇的形式变体。
4. 番剧只是来源，不是知识分类主轴。
5. 功能是主分类，语体和关系距离是判断是否可迁移的关键。
6. 用户声明掌握程度时，应作为有效证据写入更新记录。
7. 快速路由索引用于节省上下文，详细条目用于必要时展开。
8. 同一表达簇跨番剧复现时，合并来源、练习记录和复习状态，不创建重复进度。
9. 快速路由索引只是导航缓存，正文条目才是事实来源。
10. 普通学习时优先读取下次判断、出题和合并所需的信息；低频历史和完整明细默认不展开。

## 掌握状态定义

| 状态 | 含义 | 典型证据 |
|---|---|---|
| 接触 | 系统讲过并布置练习，但尚未验证 | 静默练习；刚学完 |
| 可识别 | 能听懂或看懂 | 理解题通过；用户声明已经懂了 |
| 提示下可复现 | 目标表达刚展示过时，能套用或改写出来 | 看过候选清单/讲解后当场输出通过 |
| 可复现 | 不看答案和目标表达时，能在原场景或近似场景说出来 | 原场景延迟复习通过；用户声明会说 |
| 可迁移 | 隔一段时间后，不展示目标表达，换场景仍能自然使用 | 延迟迁移题通过；用户声明可换场景使用 |
| 稳定 | 多次跨场景、跨番剧仍能自然使用 | 多轮验证；用户声明很熟 |
| 豁免 | 不进入主动训练或以后不用考 | 用户要求跳过；只需听懂 |

## 现实可用度

| 等级 | 含义 |
|---|---|
| A | 现实中自然常用，优先练成主动表达 |
| B | 现实中可用，但要注意对象、语气或场景 |
| C | 能听懂即可，主动使用需谨慎 |
| D | 主要作为理解材料，不建议主动模仿 |

## 读取分层规则

本文件保持单文件结构，但内部按读取频率分层，避免每次学习都展开完整历史。

普通学习默认只读：

1. `1. 快速路由索引`
2. `2. 活跃复习摘要`

只有以下情况才读取后面的详细内容：

- 快速路由索引无法判断是否属于已有表达簇。
- 候选表达命中某个表达簇，需要确认具体变体、错因或待验证点。
- 本轮要写回、升级、降级、合并来源或处理用户声明。
- 用户进入复习、统计、迁移或资源维护模式。

`4. 低频归档` 和 `5. 历史明细归档` 默认不参与普通学习读取；除非相关表达重新出现、用户明确询问，或进入资源维护/统计模式。

Agent 每次处理字幕时，先看本区。只有快速路由索引无法判断时，才展开详细条目。

路由类型：

- `主动表达`：现实可用，值得主动说。

### 轻量写入规则

为了节省 token，新增表达簇默认使用轻量记录，不必每次填满所有小节。

必填内容：

- 核心意义、主功能、检索键、状态、现实可用度、关系距离、语气说明。
- 复习状态摘要中的当前阶段、最近验证、待验证点、复习策略。
- 一行字幕上下文记录：番剧、集数、原台词、谁对谁说、刚发生什么、互动功能。
- 形式变体只记录本轮真正练过或下次需要考的形式。

选填内容：

- 易混表达：只有用户出错或确实容易混时才写。
- 练习记录：只保留有升级、降级、错因或用户声明价值的记录。
- 更新记录：只保留状态变化，不记录普通引用。
- 现实可用版本：如果形式变体已经覆盖，可省略。

已有详细条目不自动压缩；压缩旧条目必须进入资源维护模式并等待用户确认。

## 状态更新规则

### 新表达

当字幕中出现尚未覆盖的新核心功能表达：

1. 新增表达簇详细条目。
2. 写入快速路由索引。
3. 如需近期验证，写入活跃复习摘要或 `04-复习队列.md`。
4. 写入按功能索引。
5. 写入番剧来源索引。
6. 若只是静默练习，状态最多为 `接触`。
7. 建立“复习状态摘要”，证据强度默认为弱，最近有效接触写入本轮日期和来源。

### 已有表达的新变体

当字幕中出现已有表达簇的礼貌形、简体、缩略、方言、角色口癖等：

1. 不新建表达簇。
2. 在原表达簇的“形式变体”中追加。
3. 在“来源记录”中追加番剧和原台词。
4. 根据变体熟悉程度设置变体状态。
5. 如果用户说“核心意思我会，只是不熟这个说法”，表达簇状态不降级，只把该变体加入复习。
6. 更新复习状态摘要中的最近有效接触和待验证点。

### 已有表达跨番剧复现

当同一表达簇在新番剧或新集数中再次出现：

1. 不新建表达簇。
2. 在“来源记录”和“番剧来源索引”追加来源。
3. 如果该复现被选中教学或复习，更新最近有效接触。
4. 如果用户完成验证，更新最近验证、证据强度、当前阶段和待验证点。
5. 如果还需要复习，只在 `04-复习队列.md` 保留一个指向该表达簇的候选项，不创建独立进度。

### 用户声明

当用户声明掌握程度：

1. 调整表达簇状态或变体状态。
2. 更新复习状态摘要中的当前阶段、证据强度、最近验证和待验证点。
3. 在练习记录或更新记录写明“根据用户声明”。
4. 如有必要，从 `04-复习队列.md` 移除或降低优先级。

### 用户自我修正

当用户说“这个其实我不会说”“我以为会但说不出来”等自我修正时：

1. 尊重用户声明，允许表达簇或变体降级。
2. 若原状态是 `稳定` 或 `可迁移`，可降为 `可识别` 或 `接触`。
3. 证据强度改为弱或中。
4. 在更新记录中写明“用户自我修正”。
5. 加入 `04-复习队列.md`，下次用原场景复现或迁移题验证。

### 错误后的保守更新

当用户回答错误：

1. 不升级表达簇状态。
2. 必要时降低对应变体状态。
3. 在更新记录中写明具体错因。
4. 更新复习状态摘要中的最近验证、证据强度和待验证点。
5. 保留或加入复习队列。

### 有效接触与验证

不记录每次普通引用，只记录学习事件。

有效接触包括：正式讲解、布置练习、旧表达复现被选中复习、新语体/方言/关系距离/功能变体出现。

验证包括：用户回答理解题、复现题或迁移题，用户答错，用户声明掌握程度或要求跳过。

## 合并与去重规则

多次学习、多部番剧后，可能出现相似表达。按以下标准判断是否合并。合并不是按字面决定，而是按核心意义、主功能、语气和关系距离决定。

### 应合并为同一表达簇

- 核心意义相同。
- 主功能相同。
- 语气和关系距离基本一致。
- 只是普通形、丁寧形、敬语、缩略或方言不同。
- 关系距离不同但只是同一功能的礼貌级、亲疏级或方言变体，且主动学习目标一致。
- 用户明确说明“这个和之前那个是同一个意思/我已经掌握”。

### 应拆成不同表达簇

- 字面相似，但功能不同。
- 字面相同，但语气、关系距离或互动功能不同。
- 同一句在不同场景中承担完全不同的互动功能。
- 一个是请求，一个是命令或警告。
- 一个是现实可用表达，另一个是角色化口癖，且不宜归为同一主动学习目标。

### 合并时保留的信息

如果两个表达簇被判定重复：

1. 保留掌握状态更高且记录更完整的条目作为主条目。
2. 把另一个条目的来源、变体、练习记录和复习状态摘要并入主条目。
3. 在更新记录中写明“合并自 EXP-xxxx”。
4. 快速路由索引和功能索引只保留主条目ID。

## 迁移兼容规则

旧版文件可能没有复习状态摘要、检索键或快速路由索引。迁移时不要要求用户补全历史细节。

默认补值：

- 当前阶段：沿用旧表达簇状态。
- 证据强度：`接触` 为弱；`可识别/可复现` 为中；`可迁移/稳定` 为强。
- 最近有效接触：未记录。
- 最近验证：未记录。
- 熟悉度状态：未确认。
- 待验证点：下次遇到相关字幕时轻量验证。

无法判断的信息保留原文，并标记为“待确认”。
