# Anime Japanese Learning System

一套面向 Codex 对话使用的番剧日语口语学习提示词文件包。

这个项目不是自动化 App，也不是字幕翻译工具。它的目标是让你每次把番剧字幕交给 Codex 时，Codex 都能按照稳定规则筛选台词、判断现实可用度、设计口语练习，并把你的学习状态沉淀到可迁移的知识库里。

## 使用场景

1. 从 GitHub 下载本项目的 6 个 Markdown 文件。
2. 在 Codex 里新开一个对话，把这些文件和字幕文件一起上传。
3. Codex 按文件中的规则进行教学。
4. 教学结束后，只需要手动保留和迁移 `05-已习得库.md`。
5. 下一次学习时，重新下载原始文件，再把上一次更新过的 `05-已习得库.md` 合并进去。

这样，每次新对话仍然是干净的，但你的长期学习成果会持续累积。

## 文件结构

```text
.
├── AGENTS.md
├── 01-学习偏好.md
├── 02-学习流程.md
├── 03-错因日志.md
├── 04-复习队列.md
└── 05-已习得库.md
```

## 每个文件的作用

| 文件 | 作用 | 是否长期继承 |
|---|---|---|
| `AGENTS.md` | 定义 Codex 的身份、规则、文件更新逻辑 | 通常不变 |
| `01-学习偏好.md` | 记录长期学习偏好 | 通常不变 |
| `02-学习流程.md` | 记录字幕处理和练习流程 | 通常不变 |
| `03-错因日志.md` | 记录本次对话观察到的错误 | 不继承 |
| `04-复习队列.md` | 记录本次对话需要复习的项目 | 不继承 |
| `05-已习得库.md` | 记录跨番剧、跨对话的表达掌握状态 | 继承 |

## 独特设计

### 1. 表达簇，而不是句子列表

日语里同一个意思可能有很多外形：普通形、丁寧形、敬语、缩略、方言、角色口癖等。

所以 `05-已习得库.md` 不把知识记录成孤立句子，而是记录成“表达簇”。例如同一个“请求许可”的核心表达，可以包含：

- `〜てもいい？`
- `〜てもいいですか`
- `〜てもええ？`
- `〜ても大丈夫？`

这样，当你已经掌握核心表达时，系统不会因为换了礼貌形或方言就把它误判成全新知识。

### 2. 用户声明优先

Codex 并不总能从一次对话里准确判断你真实掌握了什么。因此规则明确规定：

如果你说“这个我已经懂了”“这个我会说”“这个只是关西说法，我懂”，系统必须把这视为状态更新证据，而不是闲聊。

这让知识库可以被你主动校准，避免系统反复教学你已经会的内容。

### 3. 静默练习默认规则

很多时候，Codex 最后让你练一句话，你会看完并练习，但不会继续回复。

本项目默认：如果系统最后布置了练习，而你没有继续回复，就视为你完成了一次静默练习。但状态更新必须保守，最多只能标为 `接触`，不能自动升级为掌握。

这既符合真实使用习惯，也避免夸大学习结果。

### 4. 跨番剧迁移

传统番剧学习很容易变成：

```text
看一部番 → 记一些句子 → 换一部番 → 重新开始
```

这个项目的目标是：

```text
看一部番 → 筛选表达 → 练习输出 → 写入表达簇 → 换番继续继承
```

多部番剧之后，知识库会记录：

- 你已经掌握哪些核心功能。
- 哪些表达只是新变体。
- 哪些说法适合现实中主动使用。
- 哪些只需要听懂。
- 哪些内容下次不用重复教。

### 5. 节省 Token 的快速索引

`05-已习得库.md` 设计了“快速索引”。

Codex 每次处理字幕时，先看快速索引，只在无法判断时再展开详细条目。这样可以减少每次对话读取整份知识库的成本，尤其适合知识库逐渐变大后的长期使用。

## 推荐工作流

1. 新对话开始时，上传本项目文件。
2. 如果你已经有上一轮更新后的 `05-已习得库.md`，用它替换原始版本。
3. 上传本集字幕。
4. 让 Codex 按 `AGENTS.md` 开始教学。
5. 教学结束时，把 Codex 输出的状态更新合并到 `05-已习得库.md`。
6. 下次学习只迁移更新后的 `05-已习得库.md`。

## 适合谁

这个项目适合：

- 想用番剧练日语口语的人。
- 不想逐句翻译，而想筛选现实可用表达的人。
- 希望跨番剧积累表达能力的人。
- 希望 Codex 在新对话里仍然“记得自己学过什么”的人。
- 希望节省上下文和 token，而不是每次重新解释学习偏好的人。

## 注意

本项目依赖 Codex 对上传文件的阅读和执行能力。它本身不包含自动下载字幕、自动同步 GitHub 或自动合并知识库的脚本。

`05-已习得库.md` 的迁移和合并目前设计为手动完成，这是为了保持系统轻量、透明，并避免自动化过度复杂。
