# Anime Japanese Learning System

一套面向 Codex 对话使用的番剧日语口语学习提示词文件包。

这个项目不是自动化 App，也不是字幕翻译工具。它的目标是让你每次把番剧字幕交给 Codex 时，Codex 都能按照稳定规则筛选台词、判断现实可用度、设计口语练习，并把你的学习状态沉淀到可迁移的全局口语记忆里。

## v2 更新声明：多番剧并行与对话式动态复习

v2 将系统从“单番剧学习项目”升级为“多番剧输入下的全局口语学习系统”。

现在，多部番剧可以共用同一套学习记忆。番剧只是素材来源，不是知识边界；表达簇、错因、复习候选和掌握进度都服务于用户整体日语口语能力。

本次更新的重点：

- `03-错因日志.md` 和 `04-复习队列.md` 在一个学习周期内跨 Codex 新对话保留，直到用户手动清空。
- `04-复习队列.md` 不再是每日任务表，而是动态复习候选池。
- `05-已习得库.md` 是全局真实进度来源，记录表达簇、变体、来源和复习状态摘要。
- 同一表达簇跨番剧出现时，合并来源和复习进度，不创建重复条目。
- 复习算法不套用 Anki 固定间隔，而是在每次对话中根据当前字幕、错因、证据强度和认知负担选择少量复习项。
- 系统通过快速索引和低精度事件记录节省 token：不记录每次普通引用，只记录讲解、练习、回答、答错、用户声明、新变体和旧表达复现等学习事件。

## 使用场景

1. 从 GitHub 下载本项目的 Markdown 文件。
2. 在 Codex 里新开一个对话，把这些文件和字幕文件一起上传。
3. Codex 按文件中的规则进行教学。
4. 教学结束后，保留本学习周期更新后的 `03-错因日志.md`、`04-复习队列.md` 和 `05-已习得库.md`。
5. 下一次学习时，重新下载原始文件，再把当前学习周期的 `03/04/05` 合并进去。

这样，每次新对话仍然能轻量启动，但你的错因、复习候选和长期掌握进度会持续累积。

## 每个文件的作用

| 文件 | 作用 | 是否继承 |
|---|---|---|
| `AGENTS.md` | 定义 Codex 的身份、规则、文件更新逻辑 | 通常不变 |
| `01-学习偏好.md` | 记录长期学习偏好 | 通常不变 |
| `02-学习流程.md` | 记录字幕处理和练习流程 | 通常不变 |
| `03-错因日志.md` | 记录当前学习周期的全局错因 | 周期内继承 |
| `04-复习队列.md` | 记录当前学习周期的动态复习候选池 | 周期内继承 |
| `05-已习得库.md` | 记录跨番剧、跨对话的表达簇和真实掌握进度 | 长期继承 |

## 独特设计

### 1. 表达簇，而不是句子列表

日语里同一个意思可能有很多外形：普通形、丁寧形、敬语、缩略、方言、角色口癖等。

所以 `05-已习得库.md` 不把知识记录成孤立句子，而是记录成“表达簇”。例如同一个“请求许可”的核心表达，可以包含：

- `〜てもいい？`
- `〜てもいいですか`
- `〜てもええ？`
- `〜ても大丈夫？`

这样，当你已经掌握核心表达时，系统不会因为换了礼貌形或方言就把它误判成全新知识。

### 2. 用户声明优先

Codex 并不总能从一次对话里准确判断你真实掌握了什么。因此规则明确规定：

如果你说“这个我已经懂了”“这个我会说”“这个只是关西说法，我懂”，系统必须把这视为状态更新证据，而不是闲聊。

这让知识库可以被你主动校准，避免系统反复教学你已经会的内容。

### 3. 静默练习默认规则

很多时候，Codex 最后让你练一句话，你会看完并练习，但不会继续回复。

本项目默认：如果系统最后布置了练习，而你没有继续回复，就视为你完成了一次静默练习。但状态更新必须保守，最多只能标为 `接触`，不能自动升级为掌握。

这既符合真实使用习惯，也避免夸大学习结果。

### 4. 多番剧并行

传统番剧学习很容易变成：

```text
看一部番 → 记一些句子 → 换一部番 → 重新开始
```

这个项目的目标是：

```text
多部番剧 → 筛选表达 → 练习输出 → 写入全局表达簇 → 合并复习进度
```

多部番剧之后，知识库会记录：

- 你已经掌握哪些核心功能。
- 哪些表达只是新变体。
- 哪些说法适合现实中主动使用。
- 哪些只需要听懂。
- 哪些内容下次不用重复教。

同一表达簇在不同番剧中出现时，系统会追加来源记录并合并复习进度；如果字面相同但功能、语气或关系距离不同，则拆成不同表达簇。

### 5. 对话式动态复习

`04-复习队列.md` 不是每日任务表。它是 Agent 在每次对话前查看的动态候选池。

有新字幕时，系统只插入 1-3 个与当前字幕相关的旧表达复习；没有新字幕、用户想复习时，再从候选池中选择 3-5 个重点。复习选择会考虑：

- 当前字幕相关度。
- 历史错因。
- 证据强度。
- 待验证点。
- 现实可用度。
- 当前对话负担。
- 用户声明掌握程度。

### 6. 节省 Token 的快速索引和轻量事件记录

`05-已习得库.md` 设计了“快速索引”。

Codex 每次处理字幕时，先看快速索引，只在无法判断时再展开详细条目。这样可以减少每次对话读取整份知识库的成本，尤其适合知识库逐渐变大后的长期使用。

系统也不会记录每次普通引用，只记录真正有学习意义的事件，例如正式讲解、布置练习、用户回答、答错、用户声明、新变体和旧表达复现。

## 推荐工作流

1. 新对话开始时，上传本项目文件。
2. 如果你已经有当前学习周期更新后的 `03/04/05`，用它们替换原始版本。
3. 上传本集字幕。
4. 让 Codex 按 `AGENTS.md` 开始教学。
5. 教学结束时，把 Codex 输出的状态更新合并到 `03/04/05`。
6. 当一个学习周期结束后，可以手动清空 `03/04`，但保留 `05` 作为长期资产。

## 适合谁

这个项目适合：

- 想用番剧练日语口语的人。
- 不想逐句翻译，而想筛选现实可用表达的人。
- 同时看多部番剧，希望共用一套口语记忆的人。
- 希望 Codex 在新对话里仍然“记得自己学过什么”的人。
- 希望节省上下文和 token，而不是每次重新解释学习偏好的人。

## 注意

本项目依赖 Codex 对上传文件的阅读和执行能力。它本身不包含自动下载字幕、自动同步 GitHub 或自动合并知识库的脚本。

`03/04/05` 的迁移和合并目前设计为手动完成，这是为了保持系统轻量、透明，并避免自动化过度复杂。
